## 📁 推荐的“文件即数据库”目录结构

你可以建立一个中央仓库，按以下逻辑规范命名和存放：

```text
sc_repository/
  ├── species_meta.csv              # 物种信息表（记录物种名、TaxID、参考基因组版本）
  │
  ├── proteins/                     
  │   ├── human_uniprot.fa          # 人类蛋白质序列 (FASTA)
  │   └── mouse_uniprot.fa          # 小鼠蛋白质序列 (FASTA)
  │
  ├── cell_markers/                 
  │   ├── human_pbmc_GSE123_markers.csv  # 某个数据集算出来的细胞特异 Marker 表 (CSV)
  │   └── mouse_brain_GSE456_markers.csv
  │
  └── expression_matrices/          
      ├── human_pbmc_GSE123.h5ad    # 单细胞表达矩阵核心文件 (H5AD)
      └── mouse_brain_GSE456.h5ad

```

## 💡 为什么这三个文件足够好？（三维再评价）

* **资源节约（★★★★★）**：
* **0 内存占用**：不运行、不查询时，它们只是硬盘里静静躺着的文件，不消耗一丁点服务器 CPU 和内存。
* **极致压缩**：`.h5ad` 内部自带高效的稀疏矩阵压缩（Scipy Sparse Matrix），比任何 SQL 数据库都省空间。


* **调用难易度（★★★★★）**：
* **各司其职**：用 `scanpy` 读 H5AD，用 `pandas` 读 CSV，用 `Biopython` 读 FASTA。全都是生信分析师闭着眼都会写的代码。


* **维护难易度（★★★★★）**：
* **绝对自由**：想备份？整个文件夹打包带走。想删数据？直接右键删除。不需要担心任何数据库配置崩掉。

## 🛠️ 进阶小技巧：如何用 Python 优雅地“秒级”调用它们？

既然不用 SQL 数据库，我们就要利用好这三种文件的特长：

### 1. 读 H5AD 矩阵时：开启“懒加载”模式（极省内存）

如果一个 H5AD 文件有好几个 GB，你只是想进去查一下某个基因的表达量，千万不要把它全读进内存！

```python
import scanpy as sc

# 使用 backed="r"（只读指针模式），这行代码执行只需 0.1 秒，不吃内存！
adata = sc.read_h5ad("expression_matrices/human_pbmc_GSE123.h5ad", backed="r")

# 只有在你真正切片提取特定基因时，才会去硬盘读对应的数字
cd3d_expression = adata[:, "CD3D"].X.toarray()

```

### 2. 查特定基因序列时：给 FASTA 文件建立“索引”

不需要每次都用循环去遍历整个几百 MB 的 `proteins.fa` 文件。使用 `Biopython` 的 `FastaToDict` 或 `bgzip` 索引，可以实现像数据库一样的秒级查询：

```python
from Bio import SeqIO

# 建立一个虚拟字典索引（第一次慢，以后瞬间定位）
record_dict = SeqIO.index("proteins/human_uniprot.fa", "fasta")

# 像查字典一样，直接秒级获取某个蛋白 ID 的序列
print(record_dict["P04234"].seq)

```

这样的管理方式，既保持了你熟悉的生信文件格式，又具备了数据库的条理。只要规范好文件名和目录，一个人或小团队用起来绝对爽快！
