## 蛋白质预测与序列处理 QA 速查

**Q1: 新测序物种的蛋白质序列是怎么来的？**
**A:** 两条路径：
1.  **转录组路线（更准）**：RNA-Seq测序 → 组装成转录本（Trinity） → **TransDecoder**找编码区 → 翻译得蛋白。
2.  **基因组路线**：DNA测序 → 组装 → 用 **Augustus**（需训练）或 **GeneMark-ES**（自训练，适合全新物种）预测基因 → 翻译得蛋白。

**Q2: TransDecoder的核心原理是什么？**
**A:** 它不找最长的，而是找**最像**编码区的 ORF。它通过计算 ORF 的核苷酸组成（六联体频率，马尔可夫模型）来打分，真基因的密码子使用有偏好，能和随机序列区分开。

**Q3: 新物种的密码子规则不同怎么办？**
**A:** 这正是 TransDecoder 的优势。它的打分模型是**从你输入的转录本里动态学习**出来的，会自动捕捉该物种特有的密码子偏好，无需预设密码子表。

**Q4: 蛋白质序列里有 `*` 怎么办？**
**A:**
-   **末端 `*`**：**这是正常的**，代表终止密码子。
-   **中间 `*`**：**这是错误**，可能源于测序错误或密码子表用错。
-   **解决**：分析前统一移除。
    ```bash
    sed -i 's/\*//g' your_proteins.fasta
    ```

**Q5: 序列里有 `.` 或 `-` 怎么办？**
**A:**
-   **来源**：它们都是比对产生的**空位**，不是真实氨基酸。
-   **关键禁忌**：**绝对不能用 `*` 替换 `.` 或 `-`**。`*`是终止符占位，`.`/`-`是空位，语义完全不同，替换会使序列分析出错。
-   **解决**：
    -   **不保留比对**（如用于BLAST）：直接删除。
        ```bash
        sed -i 's/\.//g; s/-//g' your.fasta
        ```
    -   **保留比对**（如用于建树）：用 **trimAl** 删除空位列。
        ```bash
        trimal -in align.fasta -out trimmed.fasta -gt 0.8
        ```

**Q6: 常用蛋白质预测翻译软件怎么选？**
**A:**
| 场景 | 推荐软件 | 关键点 |
| :--- | :--- | :--- |
| 有 RNA-Seq 的真核生物 | **TransDecoder** | 转录本预测的标杆 |
| 有基因组的原核生物 | **Prodigal** | 原核预测的黄金标准 |
| 有近缘参考的真核基因组 | **Augustus** | 必须用匹配的物种模型训练 |
| 无近缘参考的真核基因组 | **GeneMark-ES/ET** | 无监督自训练，适合全新物种 |

**Q7: 最终处理推荐流程？**
**A:**
1.  **预测**：根据物种选对软件，获得初始蛋白序列。
2.  **清理**：拿到 `fasta` 文件后，**第一步就删掉所有 `*`**。
3.  **检查**：用 `grep -n '\*'` 确认无残留星号。
4.  **比对后处理**：如果序列有 `.` 或 `-`，按是否需要保留比对信息选择直接删除或 trimAl 修剪。
